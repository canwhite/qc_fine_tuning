# PPO 损失函数 - 给 14 岁的你讲明白

## 先讲一个故事

想象你在教一个机器人写作文。

你给它一个题目，它写了一篇作文，然后你给这篇作文打分。

但是有个问题：**如果机器人只看分数，它可能会走极端**。

比如，它发现写得很长能得高分，就开始疯狂写废话。或者发现用某些词能得高分，就开始疯狂堆砌这些词。

这就是 PPO 要解决的问题：**让机器人慢慢进步，但不要走极端**。

---

## PPO 是什么？

PPO = Proximal Policy Optimization（近端策略优化）

翻译成人话：**"让策略（机器人）小步前进，别跑太快"**。

就像你学骑自行车：

- 太快了 → 容易摔倒（策略崩溃）
- 太慢了 → 学不会（训练太慢）
- PPO 就是让你保持一个安全的速度

## 整个过程分三步

```
第一步：让机器人写作文（收集数据）
    ↓
第二步：算出"这篇作文比平均好多少"（计算优势）
    ↓
第三步：调整机器人的大脑（更新参数）
```

## 第一步：让机器人写作文

### 1.1 机器人有个"大脑"

这个大脑叫 **Policy（策略）**，就是一个语言模型。

你给它题目，它生成回答：

```
题目: "什么是人工智能？"
机器人: "人工智能是让机器像人一样思考的技术..."
```

### 1.2 机器人还会"自信度"

机器人每写一个字，心里都有个"自信度"：

```
"人工智能" 这个词，我觉得有 80% 概率应该用 → logprob = -0.22（越高越自信）
"是" 这个词，我觉得有 90% 概率应该用 → logprob = -0.11
```

这个"自信度"就是 **logprob**（对数概率）。

**为什么要记录它？**

- 因为后面我们要看机器人"变自信了"还是"变不自信了"
- 如果一个好的词，机器人变得不自信了，那就糟了！

### 1.3 还有个"参照机器人"

这个参照机器人是**原始的、没训练过的版本**，它的脑子被冻住了（不学习）。

为什么要它？

- 防止新机器人变得太离谱
- 就像你学做饭，不能把原来的"常识"都丢了

### 1.4 还有个"打分老师"

这是一个 **Reward Model（奖励模型）**，专门给作文打分。

```
题目 + 作文 → 打分老师 → "这篇作文 85 分！"
```

---

## 第二步：算出"这篇作文比平均好多少"

### 2.1 先算"奖励"

这里有个关键点：**奖励 ≠ 老师打分**

```
实际奖励 = 老师打分 - 偏离惩罚
```

**什么是"偏离惩罚"？**

就是新机器人和参照机器人写的不一样程度。

```
偏离惩罚 = KL散度 × 惩罚系数

KL散度 ≈ 新机器人写的 - 参照机器人写的 差异有多大
```

**为什么要惩罚偏离？**

想象你是个学生：

- 你可以创新，但不能胡来
- 如果你写的和原来完全不一样，那可能跑偏了
- 所以要稍微"惩罚"一下这种跑偏

**举个具体的例子**：

```
新机器人写： "人工智能是一个很酷的技术"
参照机器人写： "人工智能是模拟人类智能的技术"

两个回答差别很大 → KL散度高 → 惩罚高
实际奖励 = 老师打分 80 - 惩罚 20 = 60
```

### 2.2 再算"优势"（Advantage）

**优势 = 这篇作文比我预期的好多少**

问题是：什么是"预期"？

机器人还有一个 **Value Head（价值头）**，它能预测"这篇作文大概能得多少分"。

```
Value Head 预测：这篇作文大概 70 分
实际奖励：60 分
优势 = 60 - 70 = -10（比预期差！）
```

或者：

```
Value Head 预测：这篇作文大概 50 分
实际奖励：60 分
优势 = 60 - 50 = +10（比预期好！）
```

### 2.3 GAE：让优势更准确

GAE = Generalized Advantage Estimation

问题是：一篇作文有很多字，每个字都应该有"功劳"。

但是老师只在最后打了一个分，怎么办？

**解决方法：从后往前算功劳**

```
作文结尾那个字：功劳最大（老师刚打完分）
倒数第二个字：功劳次之
...
作文开头：功劳最小
```

**GAE 公式**（别怕，我解释每个符号）：

$$A_t = \delta_t + (\gamma\lambda)\delta_{t+1} + (\gamma\lambda)^2\delta_{t+2} + ...$$

翻译成人话：

```
第 t 个字的优势 =
    这个字的 TD误差
    + 0.95 × 下一个字的 TD误差
    + 0.95² × 下下个字的 TD误差
    + ...
```

**什么是 TD误差？**

$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$

翻译：

```
TD误差 =
    这个字得到的奖励
    + 0.99 × 下个字的预测价值
    - 这个字的预测价值
```

**简单理解**：

- TD误差就是"惊喜程度"
- 如果实际比预测好，TD误差为正
- 如果实际比预测差，TD误差为负

## 第三步：调整机器人的大脑

现在我们知道每个词的"优势"了，该调整机器人了！

### 3.1 核心问题：调整多少？

机器人已经写完作文了，现在要回顾：

```
这个词我写了，优势是 +5（好事）→ 以后多写这个词！
那个词我写了，优势是 -3（坏事）→ 以后少写那个词！
```

但是！**不能调整太猛**！

### 3.2 PPO 的"刹车机制"：Clip

**关键公式来了**：

$$L^{CLIP} = -\min(r_t \cdot A_t, \text{clip}(r_t, 1-\varepsilon, 1+\varepsilon) \cdot A_t)$$

别慌，我一个个拆解：

**1. $r_t$ 是什么？**

$$r_t = \frac{\pi_{new}(a_t|s_t)}{\pi_{old}(a_t|s_t)}$$

翻译：

```
r_t = 新概率 / 旧概率
```

举例：

```
旧机器人：我觉得这个词有 50% 概率对 → 旧概率 = 0.5
新机器人：我觉得这个词有 60% 概率对 → 新概率 = 0.6
r_t = 0.6 / 0.5 = 1.2（概率增加了 20%）
```

**2. $\text{clip}(r_t, 1-\varepsilon, 1+\varepsilon)$ 是什么？**

这是 PPO 的核心！"clip"就是"裁剪"。

```
如果 r_t > 1 + ε（比如 > 1.2），就把它裁剪成 1.2
如果 r_t < 1 - ε（比如 < 0.8），就把它裁剪成 0.8
否则保持不变
```

**为什么要裁剪？**

举例：

```
假设 ε = 0.2（20%）

情况1：优势为正（A = +5），但概率变化太大
- r_t = 2.0（概率翻倍了，太猛！）
- 裁剪后 r_t = 1.2
- 损失 = -1.2 × 5 = -6（没被太奖励）

情况2：优势为正（A = +5），概率变化合理
- r_t = 1.1（概率增加了 10%，合理）
- 不裁剪
- 损失 = -1.1 × 5 = -5.5（被正常奖励）
```

**一句话总结**：如果你一步走太远，我就假装你没走那么远，不给你那么多奖励。

**3. $\min$ 是什么意思？**

取两个里面较小的那个。

```
两个损失：
- 不裁剪版本：r_t × A_t
- 裁剪版本：clip(r_t) × A_t

取较小的 → 更保守 → 不会过度优化
```

**4. 为什么有个负号？**

```
我们是在最小化损失，但目标是最大化奖励
所以损失 = -奖励
```

### 3.3 完整的策略损失例子

假设：

- 优势 A = +10（这个词是好事）
- 旧概率 = 0.5
- 新概率 = 0.8 → r_t = 1.6
- ε = 0.2

**不裁剪版本**：

```
损失 = -r_t × A = -1.6 × 10 = -16
```

**裁剪版本**：

```
r_t = 1.6 被裁剪成 1.2
损失 = -1.2 × 10 = -12
```

**取 min**：

```
最终损失 = min(-16, -12) = -16
```

等等，-16 比 -12 小，为什么取 -16？

哦，我搞错了！min 是对 **损失值** 取 min，但负数的大小关系是反过来的。

实际上代码里是：

```
pg_losses = -A × r         # 无裁剪
pg_losses2 = -A × clip(r)  # 裁剪版
pg_loss = max(pg_losses, pg_losses2)
```

因为损失要最小化，所以取 max 更保守（更接近 0，惩罚更小）。

## 还有两个损失

### 价值损失（Value Loss）

机器人有个"预测分数的能力"（Value Head），我们要训练它预测得更准。

$$L^{VF} = \frac{1}{2}(V(s) - R)^2$$

翻译：

```
价值损失 = 0.5 × (预测分数 - 实际分数)²
```

这就是简单的平方误差，让它预测越来越准。

### 总损失

$$L_{total} = L^{policy} + c_1 \cdot L^{value}$$

翻译：

```
总损失 = 策略损失 + 系数 × 价值损失
```

## 关键词详解：是什么、怎么算、从哪来

这一部分详细解释 PPO 中的每个关键概念，包括它们的定义、计算方式和来源。

### 1. logprob（对数概率）

#### 是什么？

logprob 是模型对"生成某个词"的自信度的对数形式。

$$\text{logprob} = \log P(\text{词} | \text{前面的内容})$$

#### 为什么用对数？

- 概率是 0~1 之间的数，对数后变成负数
- 方便计算（概率相乘变成对数相加）
- 数值稳定（避免很小的概率导致数值下溢）

#### 如何获取？

```python
# PyTorch 代码
logits = model(input_ids).logits              # 模型输出 logits
probs = F.softmax(logits, dim=-1)             # 转成概率
logprobs = F.log_softmax(logits, dim=-1)      # 直接算对数概率

# 取对应词的对数概率
token_logprob = logprobs[位置, token_id]
```

#### 具体例子

```
模型输出 logits: [2.0, 1.0, 0.5]（三个词的得分）

转成概率:
  softmax([2.0, 1.0, 0.5]) = [0.659, 0.242, 0.099]

对数概率:
  log([0.659, 0.242, 0.099]) = [-0.417, -1.417, -2.313]

如果选了第一个词: logprob = -0.417
```

#### 在 PPO 中的用途

- 记录生成时的 logprob → 这是 $\pi_{old}$
- 更新后重新计算 logprob → 这是 $\pi_{new}$
- 两者比较看模型变化了多少

---

### 2. KL 散度（KL Divergence）

#### 是什么？

KL 散度衡量两个概率分布的差异程度。

$$KL(P || Q) = \sum_x P(x) \log \frac{P(x)}{Q(x)}$$

#### 在 PPO 中的含义

- $P$ = 新模型的输出分布
- $Q$ = 参照模型的输出分布
- KL 散度越大 → 新模型越不像参照模型 → 越危险

#### 如何计算？

**方法 1：k1 估计器（简单）**

$$KL \approx \log \pi_{ref} - \log \pi_{new}$$

```python
kl = ref_logprobs - new_logprobs
```

**方法 2：k3 估计器（更准确）**

$$KL \approx \frac{\pi_{ref}}{\pi_{new}} - 1 - \log \frac{\pi_{ref}}{\pi_{new}}$$

```python
ratio = (ref_logprobs - new_logprobs).exp()  # π_ref / π_new
kl = ratio - 1 - (ref_logprobs - new_logprobs)
```

#### 在 PPO 中的作用

```
KL 惩罚 = -kl_coef × KL散度

实际奖励 = RM分数 + KL惩罚
```

**为什么是负的？**

- KL 大 → 偏离多 → 惩罚应该是负的（扣分）
- 所以 KL惩罚 = -kl_coef × KL

### 3. Value Head（价值头）

#### 是什么？

Value Head 是附加在语言模型上的一个**线性层**，用于预测"从当前状态开始能得多少分"。

#### 结构

```
语言模型 (Transformer)
    │
    ├── LM Head → logits → 预测下一个词（策略）
    │
    └── Value Head → 标量 → 预测未来累积奖励（价值）
```

#### 如何实现？

```python
class ModelWithValueHead(nn.Module):
    def __init__(self, base_model):
        self.base_model = base_model
        hidden_size = base_model.config.hidden_size

        # Value Head 就是一个简单的线性层
        self.value_head = nn.Linear(hidden_size, 1)

    def forward(self, input_ids):
        # 获取隐藏状态
        outputs = self.base_model(input_ids, output_hidden_states=True)
        hidden_states = outputs.hidden_states[-1]  # 最后一层的隐藏状态

        # 计算价值
        values = self.value_head(hidden_states).squeeze(-1)

        return outputs.logits, values
```

#### 输出什么？

```
输入: "什么是人工智能？" + 生成的回答
输出: 每个位置的 value 值

位置 0: value = 0.5  （预测从这里开始能得 0.5 分）
位置 1: value = 0.8  （预测从这里开始能得 0.8 分）
...
位置 T: value = 1.2  （预测从这里开始能得 1.2 分）
```

#### 训练目标

让 Value Head 的预测越来越准：
$$L_{value} = (V(s) - \text{实际回报})^2$$

### 4. Reward Model（奖励模型）

#### 是什么？

奖励模型是一个训练好的模型，专门给"问题+回答"打分。

#### 结构

```
输入: "问题" + "回答"
      ↓
   编码器 (如 DeBERTa)
      ↓
   分类头
      ↓
输出: 标量分数（越高越好）
```

#### 如何训练？

用成对比较数据训练：

```
问题: "什么是 AI？"
好回答: "AI 是人工智能..."
坏回答: "AI 就是 AI"

训练目标: 让模型给好回答打分 > 给坏回答打分
```

损失函数：
$$L = -\log\sigma(\text{score}_{好} - \text{score}_{坏})$$

#### 在 PPO 中的使用

```python
# 输入: 问题 + 回答
input_text = f"Human: {query}\n\nAssistant: {response}"

# 获取分数
with torch.no_grad():
    outputs = reward_model(input_text)
    score = outputs.logits[0, 0].item()  # 取标量分数
```

#### 常用开源 Reward Model

| 模型                                              | 语言   | 说明     |
| ------------------------------------------------- | ------ | -------- |
| OpenAssistant/reward-model-deberta-v3-large-v2    | 英文   | 常用     |
| Ray2333/reward-model-reward-model-english-chinese | 中英文 | 支持中文 |

### 5. 参数详解：gamma、lambda、beta、epsilon

#### gamma (γ) - 折扣因子

**是什么？**

- 控制未来奖励的重要性
- 越大越重视未来，越小越重视眼前

**公式中的作用**：
$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$

**取值范围**：0 ~ 1

- γ = 0：只看当前奖励（短视）
- γ = 1：同等看待所有未来奖励（远视）

**PPO 中常用值**：γ = 1.0（或 0.99）

**为什么设为 1.0？**

- 文本生成长度有限（不像游戏可以无限玩）
- 所以通常 γ = 1.0，不折扣未来奖励

#### lambda (λ) - GAE 平滑参数

**是什么？**

- 控制 GAE 中多步 TD 误差的混合程度
- 平衡偏差和方差

**公式中的作用**：
$$A_t = \delta_t + (\gamma\lambda)\delta_{t+1} + (\gamma\lambda)^2\delta_{t+2} + ...$$

**取值范围**：0 ~ 1

- λ = 0：只用单步 TD 误差（低方差，高偏差）
- λ = 1：用完整轨迹（高方差，低偏差）

**PPO 中常用值**：λ = 0.95

**怎么选？**

```
λ = 0.95 是个好折中：
- 不太平滑（避免高方差）
- 也不太急躁（避免高偏差）
```

#### beta (kl_coef) - KL 惩罚系数

**是什么？**

- 控制 KL 散度惩罚的强度
- 相当于"偏离参照模型的代价"

**公式中的作用**：
$$\text{KL惩罚} = -\beta \times KL$$

**取值范围**：通常 0.01 ~ 0.5

- β 小：允许模型大幅偏离（激进）
- β 大：强迫模型接近参照（保守）

**PPO 中常用值**：

- 初始值：0.2
- 可能自适应调整

**自适应 KL**：

```python
# 如果 KL 太大，增加 β（更保守）
if kl > target_kl * 2:
    beta *= 2
# 如果 KL 太小，减少 β（更激进）
elif kl < target_kl / 2:
    beta /= 2
```

#### epsilon (ε) - Clip 范围

**是什么？**

- PPO Clip 的边界
- 控制策略更新的步长上限

**公式中的作用**：
$$\text{clip}(r, 1-\varepsilon, 1+\varepsilon)$$

**取值范围**：通常 0.1 ~ 0.3

- ε = 0.1：概率最多变化 ±10%（保守）
- ε = 0.2：概率最多变化 ±20%（适中）
- ε = 0.3：概率最多变化 ±30%（激进）

**PPO 中常用值**：ε = 0.2

**怎么选？**

```
任务简单 → ε 可以大一点（如 0.3）
任务复杂 → ε 小一点（如 0.1）
默认 → ε = 0.2
```

### 6. ratio（概率比）

#### 是什么？

新旧策略对同一个动作的概率之比。

$$r_t = \frac{\pi_{new}(a_t|s_t)}{\pi_{old}(a_t|s_t)}$$

#### 如何计算？

```python
# log 形式更稳定
log_ratio = new_logprobs - old_logprobs
ratio = torch.exp(log_ratio)
```

#### 含义

| ratio 值 | 含义                 |
| -------- | -------------------- |
| r = 1.0  | 概率没变             |
| r = 1.2  | 概率增加了 20%       |
| r = 0.8  | 概率减少了 20%       |
| r = 2.0  | 概率翻倍了（太猛！） |

#### 在 PPO 中的作用

```
如果 ratio 太大（如 2.0）→ 被 clip 到 1.2
如果 ratio 太小（如 0.5）→ 被 clip 到 0.8
```

### 7. Reference Model（参照模型）

#### 是什么？

训练开始时的原始模型，冻结不更新。

#### 为什么要它？

1. **计算 KL 散度**：知道新模型偏离了多少
2. **防止灾难性遗忘**：保证新模型不会把原来的能力丢光
3. **作为锚点**：提供一个稳定的参照

#### 如何创建？

```python
from trl import create_reference_model

# 方法 1：复制一份
ref_model = create_reference_model(policy_model)

# 方法 2：加载同一个模型两次
policy_model = AutoModelForCausalLM.from_pretrained(model_name)
ref_model = AutoModelForCausalLM.from_pretrained(model_name)
ref_model.eval()  # 冻结
for param in ref_model.parameters():
    param.requires_grad = False
```

#### 内存优化

如果内存不够，可以让 policy 和 reference 共享大部分参数：

```python
# 只在需要时计算 reference 的输出
with torch.no_grad():
    ref_output = ref_model(input_ids)
```

## 总结：一句话理解每个概念

| 概念                  | 一句话理解                           |
| --------------------- | ------------------------------------ |
| **Policy（策略）**    | 机器人的"大脑"，决定写什么词         |
| **logprob**           | 机器人写每个词的"自信度"（对数形式） |
| **Reference Model**   | "参照机器人"，冻结不学，防止跑偏     |
| **Reward Model**      | "打分老师"，给作文打分               |
| **KL 散度**           | 新旧模型输出分布的差异程度           |
| **奖励**              | 老师分数 - KL惩罚（偏离代价）        |
| **Value Head**        | 线性层，预测从当前开始能得多少分     |
| **TD 误差**           | "惊喜程度" = 实际 - 预期             |
| **优势（Advantage）** | 比预期好多少，用 GAE 从后往前算      |
| **GAE**               | 综合多步 TD 误差，更准确的优势估计   |
| **ratio (r_t)**       | 新概率 / 旧概率，看概率变化了多少    |
| **Clip**              | "刹车"，限制 ratio 变化太快          |
| **gamma (γ)**         | 未来奖励折扣因子，通常设为 1.0       |
| **lambda (λ)**        | GAE 平滑参数，通常设为 0.95          |
| **beta**              | KL 惩罚系数，控制偏离代价            |
| **epsilon (ε)**       | Clip 范围，限制更新步长              |
| **策略损失**          | 根据优势调整概率，但不要太猛         |
| **价值损失**          | 让 Value Head 预测越来越准           |

| 参数         | 符号    | 推荐值      | 作用                   |
| ------------ | ------- | ----------- | ---------------------- |
| 折扣因子     | γ       | 1.0         | 文本生成通常不折扣     |
| GAE 参数     | λ       | 0.95        | 平衡偏差和方差         |
| KL 系数      | β       | 0.2         | 初始值，可能自适应调整 |
| Clip 范围    | ε       | 0.2         | 限制概率变化在 ±20%    |
| 学习率       | lr      | 1e-5 ~ 5e-6 | PPO 用很小的学习率     |
| 价值损失系数 | vf_coef | 0.5         | 价值损失的权重         |

## 最后：为什么 PPO 厉害？

传统方法的问题：

- 如果一步更新太猛 → 策略崩溃，机器人变傻
- 如果一步更新太小 → 训练太慢，学不会

PPO 的解决方案：

- **Clip 机制**：允许你大胆探索，但如果走太远，我就假装你没走那么远
- **KL 惩罚**：允许你创新，但不能偏离原来的常识太远

结果：

- 可以大胆更新（学得快）
- 不会跑偏（学得稳）

就像开车：

- 以前：要么开太慢，要么开太快撞车
- PPO：有刹车，可以开快点，但不会撞车

## 核心逻辑总结

用最简单的话说：

```
1. 机器人写作文，记下自己有多自信（logprob）

2. 打分老师给作文打分，但要扣掉"跑偏"的分（KL惩罚）
   → 得到奖励

3. 算出这篇作文比预期好多少（优势 Advantage）
   → 从后往前分配功劳（GAE）

4. 调整机器人：
   - 好事（优势正）→ 增加概率
   - 坏事（优势负）→ 减少概率
   - 但不能调整太猛 → 用 Clip 刹车
```

**PPO 最聪明的地方就是 Clip**：

> 你可以进步，但如果一步走太远，我就假装你没走那么远，不给你那么多奖励。这样你既能学得快，又不会跑偏。

## 公式速查表

### 1. 奖励计算

$$\text{reward}_t = -\beta \cdot KL_t \quad \text{(每个字)}$$
$$\text{reward}_T = -\beta \cdot KL_T + \text{RM\_score} \quad \text{(最后一个字加 RM 分数)}$$

**人话**：每个字都扣点分（因为偏离了参照），最后加上老师打的分。

### 2. TD 误差

$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$

**人话**：惊喜程度 = 这个字的奖励 + 下个字的预测值 - 这个字的预测值。

### 3. GAE（优势）

$$A_t = \delta_t + (\gamma\lambda)\delta_{t+1} + (\gamma\lambda)^2\delta_{t+2} + ...$$

**人话**：优势 = 我的惊喜 + 0.95×下个字的惊喜 + 0.95²×下下个字的惊喜 + ...

### 4. 概率比

$$r_t = \frac{\pi_{new}(a_t|s_t)}{\pi_{old}(a_t|s_t)} = \exp(\log\pi_{new} - \log\pi_{old})$$

**人话**：概率变化 = 新概率 / 旧概率。用对数算更稳定。

### 5. PPO 策略损失

$$L^{CLIP} = -\min(r_t \cdot A_t, \text{clip}(r_t, 1-\varepsilon, 1+\varepsilon) \cdot A_t)$$

**人话**：损失 = -min(优势×概率变化, 优势×裁剪后的概率变化)。裁剪限制了概率变化太快。

### 6. 价值损失

$$L^{VF} = \frac{1}{2}(V(s) - R)^2$$

**人话**：预测误差的平方，让预测越来越准。

### 7. 总损失

$$L = L^{CLIP} + c_1 \cdot L^{VF}$$

**人话**：策略损失 + 价值损失（带个权重）。

```
│                        第一步：收集数据                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │

│      │                                                          │
│      ├──▶ Reference（参照）──▶ ref_logprobs                     │
│      │      (冻结的原始模型)                                     │
│      │                                                          │
│      ├──▶ Value Head ──▶ values（预测分数）                     │
│      │      (线性层，预测未来奖励)                               │
│      │                                                          │
│      └──▶ Reward Model ──▶ score（老师打分）                    │
│             (训练好的打分模型)                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      第二步：计算优势                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   KL惩罚 = kl_coef × (ref_logprobs - logprobs)                  │
│          = β × KL(新模型 || 参照模型)                            │
│                                                                 │
│   奖励 = RM_score - KL惩罚（最后位置）                           │
│        = -KL惩罚（其他位置）                                     │
│                                                                 │
│   TD误差 δ_t = 奖励_t + γ×V(s_{t+1}) - V(s_t)                   │
│                                                                 │
│   优势 A_t = δ_t + γλ×δ_{t+1} + (γλ)²×δ_{t+2} + ... (GAE)       │
│                                                                 │
│   回报 R_t = A_t + V(s_t)                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      第三步：更新参数                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   多轮 PPO Epoch:                                               │
│                                                                 │
│   1. 前向传播获取新的 logprobs 和 values                         │
│                                                                 │
│   2. 计算 ratio = exp(new_logprobs - old_logprobs)              │
│                                                                 │
│   3. 如果 ratio > 1+ε → clip 到 1+ε                             │
│      如果 ratio < 1-ε → clip 到 1-ε                             │
│                                                                 │
│   4. 策略损失 = -max(A×ratio, A×clipped_ratio)                  │
│      (注意：代码用 max，因为负数关系)                            │
│                                                                 │
│   5. 价值损失 = 0.5 × (V - R)²                                  │
│                                                                 │
│   6. 总损失 = 策略损失 + vf_coef × 价值损失                      │
│                                                                 │
│   7. backward() → optimizer.step()                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 常见问题

### Q1: 为什么 PPO 要多次更新同一批数据？

**答**：PPO 是 on-policy 算法，数据用完就"过期"了。但是我们可以对同一批数据更新多次（如 4 次），提高样本效率。只要 Clip 机制在，就不会更新过头。

### Q2: Value Head 和 Reward Model 有什么区别？

|          | Value Head       | Reward Model   |
| -------- | ---------------- | -------------- |
| 输出     | 每个位置的预测值 | 整个回答的分数 |
| 训练方式 | 和 PPO 一起训练  | 提前单独训练   |
| 作用     | 用于计算优势     | 用于计算奖励   |

### Q3: 为什么 KL 惩罚只在最后一个位置加 RM 分数？

**答**：因为 Reward Model 只给整个回答打一个分。生成过程中，模型还不知道最终分数，只能用 KL 惩罚约束。到最后一个词了，才把 RM 分数加上去。

### Q4: gamma 和 lambda 有什么区别？

|          | gamma (γ)    | lambda (λ) |
| -------- | ------------ | ---------- |
| 作用     | 折扣未来奖励 | 平滑 GAE   |
| 公式位置 | TD 误差中    | GAE 累积中 |
| 典型值   | 1.0          | 0.95       |

简单理解：

- γ 决定"看重多远的未来"
- λ 决定"综合多少步的估计"

### Q5: 为什么用对数概率而不是直接用概率？

**答**：

1. **数值稳定**：概率是 0~1，相乘容易下溢；对数相加更稳定
2. **计算方便**：概率相乘变成对数相加
3. **梯度更好**：对数概率的梯度更稳定
